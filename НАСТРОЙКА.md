# Руководство по настройке игры

Простая карта: **что где лежит**, **как запустить**, и **какой файл править** под нужную задачу.

---

## 1. Как запустить и посмотреть

### Быстрая разработка (с авто-перезагрузкой, без рамки телефона)
```bash
npm install        # один раз
npm run dev        # → http://localhost:5173   (скин original)
npm run dev:solana # → скин solana
```
Игра занимает всё окно. Ориентацию можно имитировать, меняя размер окна браузера.

### Превью в рамке телефона/планшета (как в Playbox)
```bash
npm run build                              # собрать плеаблы в docs/play/
npx vite preview --outDir docs --port 5180 # → http://localhost:5180/
```
Открывается оболочка с выбором устройств (iPhone, Pixel, Galaxy, iPad…), поворотом
портрет/ландшафт, переключателем скина и звука. **Важно:** оболочка показывает
*собранную* версию — после правок в коде нужно снова `npm run build` и обновить страницу.

### Замер FPS (производительность)
Открой плеабл с `?fps` в адресе — сверху появится счётчик кадров:
```
http://localhost:5173/?fps
http://localhost:5180/play/original.html?fps
```

### Финальная сборка (то, что сдаётся)
```bash
npm run build
```
Создаёт два самодостаточных HTML-файла (всё встроено, ≤5 МБ каждый):
- `dist/index.html` — клон (original)
- `dist/solana.html` — рескин
- их же копии кладутся в `docs/play/` для публикации.

### Публикация на сайт
GitHub Pages раздаёт папку `docs/` из ветки `main`. Чтобы обновить
https://voixkz.github.io/playoff-runner/ — закоммить и запушь в `origin/main`
(после `npm run build`). Через ~1 минуту страница обновится (жёсткое обновление: Cmd+Shift+R).

## 2. Карта проекта — что за что отвечает

```
src/
  config/
    constants.ts        ← ВСЕ игровые числа (скорость, прыжок, враг, хитбоксы, физика ленты)
    skins/
      original.ts       ← скин «PayPal»: какие картинки/звуки, цвета, тексты, ссылки магазина
      solana.ts         ← скин «Solana»: то же для рескина
      types.ts          ← описание структуры скина (типы)
  core/
    App.ts              ← рендер и ПРОИЗВОДИТЕЛЬНОСТЬ (разрешение, antialias, лимит FPS)
    audio/AudioManager.ts ← воспроизведение музыки и звуков
    sdk/PlayableSDK.ts  ← кнопка установки (CTA) → магазин (MRAID/Facebook/Playbox)
    assets.ts           ← загрузка текстур и атласов
    StateMachine.ts     ← состояния игры (intro/running/pause/win/lose)
  game/
    GameController.ts    ← ГЛАВНЫЙ цикл: спавн, столкновения, очки, жизни, финиш
    Level.ts            ← СЦЕНАРИЙ уровня (что и когда появляется)
    Player.ts           ← бегун (прыжок, анимации, неуязвимость)
    entities/Enemy.ts   ← враг-преследователь
    entities/Obstacle.ts← конус-препятствие
    entities/Collectible.ts ← монетки/карты
    Parallax.ts         ← фон и деревья/кусты/фонари (бесконечная прокрутка)
    FinishLine.ts       ← финишные ворота + лента
    rope.ts             ← физика «верёвки» для падающей ленты
    Confetti.ts         ← конфетти при победе
    collision.ts        ← математика столкновений
  ui/
    Hud.ts              ← верхний интерфейс: сердечки + денежный индикатор
    Tutorial.ts         ← подсказки «Tap to start» / «Jump»
    EndCard.ts          ← финальный экран (награда + кнопка)
    Fail.ts             ← вспышка при проигрыше
    ui.css              ← ВСЕ стили интерфейса (позиции, размеры, кнопки, индикатор денег)

assets/original/        ← картинки/звуки/шрифт клона (извлечены из референса)
assets/solana/          ← картинки рескина
scripts/                ← сборка одного HTML + генерация Solana-ассетов
docs/                   ← то, что публикуется: index.html (рамка устройств) + play/ (плеаблы)
```

## 3. «Хочу изменить X» → правь этот файл

| Что изменить | Файл | Что искать |
|---|---|---|
| Скорость мира, высоту/длину прыжка | [src/config/constants.ts](src/config/constants.ts) | `WORLD.BASE_SPEED`, `PLAYER.JUMP_HEIGHT`, `JUMP_DURATION` |
| Скорость/масштаб/анимацию врага | [src/config/constants.ts](src/config/constants.ts) | блок `ENEMY` (в т.ч. `RUN_FRAME_COUNT`) |
| Жизни | [src/config/constants.ts](src/config/constants.ts) | `MAX_HP` |
| Точность столкновений (хитбоксы) | [src/config/constants.ts](src/config/constants.ts) | блок `HITBOX` |
| **Что и когда появляется на уровне** | [src/game/Level.ts](src/game/Level.ts) | массив `LEVEL_SCRIPT` |
| Денежный индикатор (вид/позиция/размер) | [src/ui/ui.css](src/ui/ui.css) | `.money-badge`, `#score-display` |
| Кнопка DOWNLOAD (форма/позиция) | [src/ui/ui.css](src/ui/ui.css) | `.footer-cta` |
| Финальный экран | [src/ui/ui.css](src/ui/ui.css) + [src/ui/EndCard.ts](src/ui/EndCard.ts) | `.end-*`, `.cta-button` |
| Тексты, валюта, цвета, ссылки магазина | [src/config/skins/original.ts](src/config/skins/original.ts) / [solana.ts](src/config/skins/solana.ts) | `theme`, `copy`, `store` |
| Заменить картинку (фон/карта/монета…) | положить файл в `assets/<skin>/` + поправить import в `skins/*.ts` | — |
| Музыку/звуки | файлы в `assets/<skin>/` + блок `audio` в `skins/*.ts` | — |
| Список устройств в превью | [docs/index.html](docs/index.html) | объект `DEVICES` |

> Размеры/позиции интерфейса заданы в **фиксированных пикселях** — поэтому индикатор
> и кнопки выглядят одинаково в портрете и ландшафте.

## 4. Производительность

Уже сделано (в [src/core/App.ts](src/core/App.ts)):
- **`antialias: false`** — убрал MSAA (главный пожиратель GPU на телефоне; для 2D-спрайтов почти не влияет на вид).
- **`ticker.maxFPS = 60`** — на экранах 120 Гц это вдвое снижает нагрузку; движение не меняется (всё считается по времени кадра).
- **`resolution` ≤ 2** — ограничение плотности пикселей.
- Спрайты фона, конфетти и сцены — переиспользуются (без пересоздания каждый кадр).

Если на слабом телефоне всё ещё тяжело — главный рычаг здесь же, в `App.ts`:
понизь `resolution` с `2` до `1.5` (станет резко легче, картинка чуть мягче).

Проверяй эффект счётчиком `?fps` (см. раздел 1).
